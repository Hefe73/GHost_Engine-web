Building a Default Demo in GHost Engine

Posted on March 11, 2026 · Tags: demo · initialization · setup · rendering

In this guide, we'll walk through building a complete default demo for GHost Engine step-by-step. You'll learn how to initialize the engine, load assets asynchronously, create game objects, and render them with a camera you can control.

Step 1: Include Required Headers

Start by including all the necessary headers for your demo. The headers provide access to the engine core, models, rendering, and input systems. 


#pragma once
#include "ghost.h"
#include "gh_model.hpp"
#include "gh_game_object.hpp"
#include "gh_job_system.hpp"
#include "gh_buffer.hpp"
#include "simple_render_system.hpp"
#include "gh_movement_controller.hpp"

Step 2: Initialize Logging and Create the Engine

Initialize the logging system and create a GHostEngine instance. This handles Vulkan instance creation, physical device selection, and logical device setup automatically. 


void main()
{
    gh::Log::Init();

    gh::GHostEngine ghost("WOW");

Step 3: Define the Global Uniform Buffer Object

Create a struct to hold data that's passed to shaders each frame. This includes the combined projection-view matrix and lighting information. 


    struct GlobalUbo
    {
        glm::mat4 projectionView{1.f};
        glm::vec3 lightDirection = 
            glm::normalize(glm::vec3{1.f, -3.f, -1.f});
    };

Step 4: Load Meshes Asynchronously with Job System

Create a job system with 8 workers. This allows you to load multiple 3D models in parallel without blocking the main thread. Use enqueue() to submit loading jobs and .get() to retrieve the results. 


    gh::GHJobSystem js{8};
    auto devicePtr = &ghost.getDevice();

    // Load first mesh asynchronously
    auto fut = js.enqueue([devicePtr]() -> std::shared_ptr<gh::GHModel>
    {
        return gh::GHModel::createModelFromFile(*devicePtr, "shaders/Mesh.obj");
    });
    std::shared_ptr<gh::GHModel> coloredCube = fut.get();

    // Load second mesh
    auto fut1 = js.enqueue([devicePtr]() -> std::shared_ptr<gh::GHModel>
    {
        return gh::GHModel::createModelFromFile(*devicePtr, "shaders/cube.obj");
    });
    std::shared_ptr<gh::GHModel> whiteCube = fut1.get();

    // Load third mesh
    auto fut2 = js.enqueue([devicePtr]() -> std::shared_ptr<gh::GHModel>
    {
        return gh::GHModel::createModelFromFile(*devicePtr, "shaders/ghost.fbx");
    });
    std::shared_ptr<gh::GHModel> colorModel = fut2.get();

Step 5: Create Game Objects from Loaded Models

Create game objects for each loaded mesh and add them to the scene. Game objects combine a model with transform data (position, rotation, scale). 


    // Create floor
    auto floor = gh::GHGameObject::createGameObject();
    floor.model = whiteCube;
    floor.transform.translation = {0.f, 2.f, 0.f};
    floor.transform.scale = {20.f, 1.0f, 20.f};
    ghost.getGameObjects().push_back(std::move(floor));

Step 6: Setup Rendering Infrastructure

Create the necessary Vulkan resources for rendering: uniform buffers, descriptor layouts, and descriptor sets. These connect your data to the GPU. 


    std::vector<std::unique_ptr<gh::GHBuffer>> uboBuffers(
        gh::GHSwapChain::MAX_FRAMES_IN_FLIGHT
    );
    ghost.buildBuffers(uboBuffers);

    auto globalSetLayout =
        gh::GHDescriptorSetLayout::Builder(ghost.getDevice())
        .addBinding(0, VK_DESCRIPTOR_TYPE_UNIFORM_BUFFER, 
                    VK_SHADER_STAGE_VERTEX_BIT)
        .build();

    std::vector<VkDescriptorSet> globalDescriptorSets(
        gh::GHSwapChain::MAX_FRAMES_IN_FLIGHT
    );
    ghost.buildGlobalDescritorsSets(
        globalDescriptorSets, 
        uboBuffers, 
        globalSetLayout
    );

    gh::SimpleRenderSystem simpleRenderSystem{
        ghost.getDevice(),
        ghost.getRenderer().getSwapChainRenderPass(),
        globalSetLayout->getDescriptorSetLayout()
    };

Step 7: Setup Camera and Input Controller

Create a camera object and position it. Then create a controller to handle keyboard input for free-flight camera movement. 


    gh::GHCamera camera{};
    auto viewerObject = gh::GHGameObject::createGameObject();
    viewerObject.transform.translation = {0.0f, -5.0f, 0.0f};
    viewerObject.transform.rotation = {0.0f, 0.0f, 0.0f};
    
    gh::KeyboardMovementController cameraController{};

Step 8: Enter the Main Render Loop

The main loop runs until the window closes. Each iteration polls input, updates the camera, and renders a frame. 


    auto currentTime = std::chrono::high_resolution_clock::now();

    while (!ghost.getWindow().shouldClose())
    {
        glfwPollEvents();

        auto newTime = std::chrono::high_resolution_clock::now();
        float frameTime = 
            std::chrono::duration<float, std::chrono::seconds::period>
            (newTime - currentTime).count();
        currentTime = newTime;

Step 9: Update Camera and Input Handling

Get the window size and update the camera using the controller. This processes keyboard input to move and rotate the camera. 


        int width, height;
        glfwGetWindowSize(ghost.getWindow().getGLFWwindow(), 
                         &width, &height);

        cameraController.flycam(
            ghost.getWindow().getGLFWwindow(), 
            frameTime, 
            viewerObject, 
            width, 
            height
        );

        camera.getView() = glm::lookAt(
            viewerObject.transform.translation,
            viewerObject.transform.translation + 
            viewerObject.transform.rotation,
            {0.0f, 1.0f, 0.0f}
        );

Step 10: Setup Camera Projection

Configure the projection matrix with a 90-degree field of view, the window's aspect ratio, and clipping planes. 


        float aspect = ghost.getRenderer().getAspectRatio();
        camera.setPerspectiveProjection(
            glm::radians(90.f), 
            aspect, 
            0.1f, 
            10000.f
        );

Step 11: Begin Frame and Setup Frame Info

Start a new frame with the renderer. If successful (buffer acquired), prepare the frame info structure that will be passed to the render system. 


        if (auto commandBuffer = ghost.getRenderer().beginFrame())
        {
            int frameIndex = ghost.getRenderer().getFrameIndex();
            gh::FrameInfo frameInfo{
                frameIndex,
                frameTime,
                commandBuffer,
                camera,
                globalDescriptorSets[frameIndex]
            };

Step 12: Update Uniform Buffer Data

Calculate the combined projection-view matrix and write it to the uniform buffer. This matrix transforms vertices from world space to screen space. 


            GlobalUbo ubo{};
            ubo.projectionView = camera.getProjection() * camera.getView();
            uboBuffers[frameIndex]->writeToBuffer(&ubo);
            uboBuffers[frameIndex]->flush();

Step 13: Render Game Objects

Record and execute render commands. Begin the render pass, render all game objects, then end the render pass and submit the frame. 


            ghost.getRenderer().beginSwapChainRenderPass(commandBuffer);
            simpleRenderSystem.renderGameObjects(
                frameInfo, 
                ghost.getGameObjects()
            );
            ghost.getRenderer().endSwapChainRenderPass(commandBuffer);
            ghost.getRenderer().endFrame();
        }

Step 14: Cleanup

After the main loop exits, wait for the device to finish all pending work before releasing resources. 


    vkDeviceWaitIdle(ghost.getDevice().device());
}

Complete Example

Here's the full main() function bringing all steps together: 


void main()
{
    gh::Log::Init();

    struct GlobalUbo {
        glm::mat4 projectionView{1.f};
        glm::vec3 lightDirection = 
            glm::normalize(glm::vec3{1.f, -3.f, -1.f});
    };

    gh::GHostEngine ghost("WOW");
    gh::GHJobSystem js{8};
    auto devicePtr = &ghost.getDevice();

    // Load meshes
    auto fut = js.enqueue([devicePtr]() -> std::shared_ptr<gh::GHModel> {
        return gh::GHModel::createModelFromFile(*devicePtr, "shaders/Mesh.obj");
    });
    auto coloredCube = fut.get();

    auto fut1 = js.enqueue([devicePtr]() -> std::shared_ptr<gh::GHModel> {
        return gh::GHModel::createModelFromFile(*devicePtr, "shaders/cube.obj");
    });
    auto whiteCube = fut1.get();

    // Create floor
    auto floor = gh::GHGameObject::createGameObject();
    floor.model = whiteCube;
    floor.transform.translation = {0.f, 2.f, 0.f};
    floor.transform.scale = {20.f, 1.0f, 20.f};
    ghost.getGameObjects().push_back(std::move(floor));

    // Setup rendering
    std::vector<std::unique_ptr<gh::GHBuffer>> uboBuffers(
        gh::GHSwapChain::MAX_FRAMES_IN_FLIGHT
    );
    ghost.buildBuffers(uboBuffers);

    auto globalSetLayout =
        gh::GHDescriptorSetLayout::Builder(ghost.getDevice())
        .addBinding(0, VK_DESCRIPTOR_TYPE_UNIFORM_BUFFER, 
                    VK_SHADER_STAGE_VERTEX_BIT)
        .build();

    std::vector<VkDescriptorSet> globalDescriptorSets(
        gh::GHSwapChain::MAX_FRAMES_IN_FLIGHT
    );
    ghost.buildGlobalDescritorsSets(
        globalDescriptorSets, uboBuffers, globalSetLayout
    );

    gh::SimpleRenderSystem simpleRenderSystem{
        ghost.getDevice(),
        ghost.getRenderer().getSwapChainRenderPass(),
        globalSetLayout->getDescriptorSetLayout()
    };

    // Setup camera
    gh::GHCamera camera{};
    auto viewerObject = gh::GHGameObject::createGameObject();
    viewerObject.transform.translation = {0.0f, -5.0f, 0.0f};
    gh::KeyboardMovementController cameraController{};

    auto currentTime = std::chrono::high_resolution_clock::now();

    while (!ghost.getWindow().shouldClose()) {
        glfwPollEvents();

        auto newTime = std::chrono::high_resolution_clock::now();
        float frameTime = 
            std::chrono::duration<float, std::chrono::seconds::period>
            (newTime - currentTime).count();
        currentTime = newTime;

        int width, height;
        glfwGetWindowSize(ghost.getWindow().getGLFWwindow(), &width, &height);

        cameraController.flycam(ghost.getWindow().getGLFWwindow(), 
                                frameTime, viewerObject, width, height);
        camera.getView() = glm::lookAt(
            viewerObject.transform.translation,
            viewerObject.transform.translation + viewerObject.transform.rotation,
            {0.0f, 1.0f, 0.0f}
        );

        float aspect = ghost.getRenderer().getAspectRatio();
        camera.setPerspectiveProjection(glm::radians(90.f), aspect, 0.1f, 10000.f);

        if (auto commandBuffer = ghost.getRenderer().beginFrame()) {
            int frameIndex = ghost.getRenderer().getFrameIndex();
            gh::FrameInfo frameInfo{frameIndex, frameTime, commandBuffer, 
                                    camera, globalDescriptorSets[frameIndex]};

            GlobalUbo ubo{};
            ubo.projectionView = camera.getProjection() * camera.getView();
            uboBuffers[frameIndex]->writeToBuffer(&ubo);
            uboBuffers[frameIndex]->flush();

            ghost.getRenderer().beginSwapChainRenderPass(commandBuffer);
            simpleRenderSystem.renderGameObjects(frameInfo, ghost.getGameObjects());
            ghost.getRenderer().endSwapChainRenderPass(commandBuffer);
            ghost.getRenderer().endFrame();
        }
    }

    vkDeviceWaitIdle(ghost.getDevice().device());
}

Key Takeaways

  • Job System: Use it to load assets in parallel without blocking rendering
  • Game Objects: Combine models with transforms to represent entities in your scene
  • Uniform Buffers: Send per-frame data like matrices to your shaders
  • Frame Loop: Poll input, update state, and render each frame
  • Camera Control: Use the built-in controller for intuitive navigation
