Vulkan Initialization in GHost Engine

Posted on March 1, 2025 · Tags: vulkan · initialization · device · instance

The first step in any Vulkan-based engine is bootstrapping the API correctly. GHost Engine handles this through a dedicated VulkanContext class that encapsulates instance creation, physical device selection and logical device setup.

Creating the Instance

We request validation layers in debug builds and enumerate the required extensions provided by the windowing system. 

VkApplicationInfo appInfo{};
appInfo.sType              = VK_STRUCTURE_TYPE_APPLICATION_INFO;
appInfo.pApplicationName   = "GHost Engine";
appInfo.applicationVersion = VK_MAKE_VERSION(0, 1, 0);
appInfo.pEngineName        = "GHost";
appInfo.apiVersion         = VK_API_VERSION_1_3;

VkInstanceCreateInfo createInfo{};
createInfo.sType            = VK_STRUCTURE_TYPE_INSTANCE_CREATE_INFO;
createInfo.pApplicationInfo = &appInfo;
vkCreateInstance(&createInfo, nullptr, &instance);

Physical Device Selection

GHost Engine scores each available GPU based on type (discrete > integrated), available queue families, and required feature support such as samplerAnisotropy and geometry shaders.

Logical Device & Queues

We request separate queue families for graphics, compute and transfer operations when available, enabling async compute passes later in the pipeline. 

VkDeviceQueueCreateInfo queueCreateInfo{};
queueCreateInfo.sType            = VK_STRUCTURE_TYPE_DEVICE_QUEUE_CREATE_INFO;
queueCreateInfo.queueFamilyIndex = indices.graphicsFamily.value();
queueCreateInfo.queueCount       = 1;
float priority = 1.0f;
queueCreateInfo.pQueuePriorities = &priority;
