Physically Based Rendering in GHost Engine

Posted on January 28, 2025 · Tags: pbr · lighting · brdf · realistic

GHost Engine implements the metallic-roughness PBR workflow popularised by glTF 2.0. The goal is to produce consistent, energy-conserving results under any lighting condition using the Cook-Torrance BRDF.

The Cook-Torrance BRDF

The specular component is computed as: f = (D · F · G) / (4 · NdotV · NdotL), where D is the normal distribution function (GGX/Trowbridge-Reitz), F is the Fresnel term (Schlick approximation), and G is the geometry/occlusion term (Smith-GGX). 

// GGX Normal Distribution Function
float D_GGX(float NdotH, float roughness) {
    float a  = roughness * roughness;
    float a2 = a * a;
    float d  = (NdotH * NdotH) * (a2 - 1.0) + 1.0;
    return a2 / (PI * d * d);
}

// Schlick Fresnel
vec3 F_Schlick(float cosTheta, vec3 F0) {
    return F0 + (1.0 - F0) * pow(1.0 - cosTheta, 5.0);
}

Material Inputs

Each material in the engine exposes four PBR parameters sampled from textures: base color (albedo), metallic factor, roughness factor, and ambient occlusion. These are packed into a UBO and bound via a descriptor set per material.

Image-Based Lighting

The indirect diffuse term uses an irradiance cubemap precomputed offline via spherical harmonic projection. The specular term uses a pre-filtered environment map combined with a BRDF LUT following the Epic split-sum approximation.
