Descriptor Sets & Uniform Buffers

Posted on January 10, 2025 · Tags: descriptors · uniforms · vulkan · memory

Descriptor sets are Vulkan's mechanism for binding resources — buffers, images, samplers — to shaders. GHost Engine uses a simple descriptor allocator with a pool-per-frame strategy to avoid per-frame allocation overhead.

Descriptor Pool

At startup we create a pool large enough for the expected maximum number of descriptor sets per frame. We reset the entire pool at the start of each frame, which is faster than freeing individual sets. 

std::vector<VkDescriptorPoolSize> poolSizes = {
    { VK_DESCRIPTOR_TYPE_UNIFORM_BUFFER,         10 },
    { VK_DESCRIPTOR_TYPE_COMBINED_IMAGE_SAMPLER, 10 },
};

VkDescriptorPoolCreateInfo poolInfo{};
poolInfo.sType         = VK_STRUCTURE_TYPE_DESCRIPTOR_POOL_CREATE_INFO;
poolInfo.poolSizeCount = static_cast<uint32_t>(poolSizes.size());
poolInfo.pPoolSizes    = poolSizes.data();
poolInfo.maxSets       = 20;
vkCreateDescriptorPool(device, &poolInfo, nullptr, &descriptorPool);

Uniform Buffer Objects

Per-frame data (view/projection matrices, light positions, time) lives in a UBO that is updated once per frame via a persistent mapped pointer, avoiding the vkMapMemory / vkUnmapMemory round trip. 

struct GlobalUBO {
    glm::mat4 view;
    glm::mat4 proj;
    glm::vec4 cameraPos;
    float     deltaTime;
};

Layout & Binding

Descriptor set layouts are created once and cached. At draw time we allocate a set from the pool, write the buffer and image descriptors, and bind the set before each draw call.
